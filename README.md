# empresa-greenenergy

## Setor
Energia Renovável

## Contexto do problema
A GreenEnergy opera soluções digitais para monitoramento de geração solar e eólica. Com o crescimento da plataforma, decisões arquiteturais importantes foram tomadas sem registro formal, reduzindo rastreabilidade de contexto, alternativas avaliadas e impactos técnicos.

## Objetivo do desafio
Produzir ADRs (Architecture Decision Records) para registrar decisões arquiteturais de forma estruturada, transparente e reutilizável.

## Instruções resumidas para os grupos
1. Faça um fork deste repositório para a conta do grupo.
2. Trabalhe exclusivamente no fork do grupo.
3. Preencha a documentação na pasta `docs/`, usando `docs/adr-template.md` como referência e consolidando o resultado em `docs/adr-001.md`.
4. Entregue o link do fork do grupo conforme orientação da disciplina.
